اصول شبکه و مفاهیم مرتبط

در دنیای شبکه‌های کامپیوتری، پروتکل‌ها و مفاهیم متعددی برای مدیریت ارتباطات و انتقال داده‌ها مورد استفاده قرار می‌گیرند. این مقاله به بررسی دقیق و کامل مفاهیم کلیدی شبکه شامل HTTP/HTTPS، DNS، فایروال، پروکسی، مدل OSI، TCP/UDP/FTP و وب‌سوکت می‌پردازد. این اطلاعات به‌گونه‌ای ارائه شده‌اند که برای افراد مبتدی تا حرفه‌ای مفید باشند.

1. HTTP/HTTPS (اعتبارنامه‌ها و امنیت وب)

HTTP (HyperText Transfer Protocol) یک پروتکل ارتباطی برای انتقال داده‌های متنی، رسانه‌ای و سایر محتواها بین مرورگر و سرور است. این پروتکل بدون وضعیت (Stateless) است، یعنی هر درخواست مستقل از سایر درخواست‌ها پردازش می‌شود و سرور اطلاعات نشست را ذخیره نمی‌کند.

ساختار درخواست و پاسخ HTTP

HTTP شامل درخواست (Request) و پاسخ (Response) است:

  • درخواست HTTP: شامل متد (مانند GET, POST)، آدرس URL، هدرها و بدنه است.
  • پاسخ HTTP: شامل کد وضعیت (200 OK, 404 Not Found)، هدرها و بدنه پاسخ است.

متدهای رایج HTTP

  • GET: دریافت اطلاعات از سرور (بارگذاری یک صفحه وب).
  • POST: ارسال داده‌ها به سرور (مانند فرم‌های ورود).
  • PUT: به‌روزرسانی یک منبع روی سرور.
  • DELETE: حذف یک منبع از سرور.
  • HEAD: مشابه GET ولی فقط هدر پاسخ را دریافت می‌کند.

HTTPS (HyperText Transfer Protocol Secure)

HTTPS نسخه امن HTTP است که از SSL/TLS برای رمزگذاری ارتباطات استفاده می‌کند و امنیت بیشتری دارد.

تفاوت HTTP و HTTPS

ویژگی HTTP HTTPS
امنیت ناامن رمزگذاری شده با SSL/TLS
استفاده در وب‌سایت‌ها بله، اما ناامن بله، استاندارد توصیه‌شده برای امنیت
محافظت در برابر حملات MITM ندارد دارد

2. DNS (سامانه نام دامنه)

DNS (Domain Name System) یک سیستم ترجمه نام دامنه به آدرس‌های IP است تا کاربران بتوانند بدون نیاز به حفظ آدرس‌های عددی، به وب‌سایت‌ها دسترسی داشته باشند.

چگونه DNS کار می‌کند؟

  1. کاربر یک نام دامنه مانند example.com را در مرورگر وارد می‌کند.
  2. مرورگر بررسی می‌کند که آیا آدرس IP مربوط به این نام دامنه در کش سیستم وجود دارد یا نه.
  3. اگر آدرس IP در کش موجود نباشد، درخواست به Recursive Resolver ارسال می‌شود.
  4. Resolver درخواست را به Root Name Server می‌فرستد.
  5. سپس، درخواست به TLD Name Server (مانند .com, .org) فرستاده می‌شود.
  6. در نهایت، Authoritative Name Server آدرس IP مربوط به دامنه را ارائه می‌دهد.

3. فایروال (Firewall)

فایروال یک سیستم امنیتی است که ترافیک ورودی و خروجی شبکه را نظارت کرده و از دسترسی‌های غیرمجاز جلوگیری می‌کند.

انواع فایروال

نوع فایروال توضیحات
Packet Filtering Firewall بررسی بسته‌های داده براساس قوانین از پیش تعیین‌شده.
Stateful Inspection Firewall بررسی وضعیت ارتباطات قبل از اجازه عبور داده‌ها.
Next-Generation Firewall (NGFW) دارای قابلیت‌های امنیتی پیشرفته مانند شناسایی تهدیدات.

4. پروکسی (Proxy Server)

پروکسی یک واسط بین کاربر و اینترنت است که درخواست‌های کاربران را به سرور اصلی ارسال کرده و پاسخ را برمی‌گرداند.

مزایای استفاده از پروکسی

مزایا توضیحات
افزایش حریم خصوصی پنهان کردن IP کاربر و افزایش امنیت.
ذخیره کش (Caching) بارگذاری سریع‌تر صفحات وب با ذخیره‌سازی داده‌ها.
عبور از محدودیت‌ها دسترسی به محتوای مسدود شده جغرافیایی.

5. مدل OSI (Open Systems Interconnection)

مدل OSI یک چارچوب استاندارد برای نحوه ارتباطات در شبکه است و از 7 لایه تشکیل شده است:

لایه نام وظایف
1 Physical انتقال بیت‌های خام از طریق کابل یا امواج.
2 Data Link مدیریت انتقال داده بین دستگاه‌های متصل به شبکه.
3 Network مسیریابی بسته‌ها و مدیریت آدرس‌های IP.
4 Transport کنترل جریان داده‌ها (TCP/UDP).
5 Session مدیریت نشست‌های ارتباطی.
6 Presentation رمزگذاری و تبدیل داده‌ها.
7 Application ارائه خدمات شبکه به کاربران (HTTP, FTP, DNS).

6. پروتکل‌های TCP, UDP, FTP

TCP (Transmission Control Protocol)

  • ارتباط قابل اطمینان و متصل‌محور.
  • دارای کنترل جریان و ترتیب بسته‌ها.
  • استفاده در وب‌گردی، ایمیل، دانلود فایل‌ها.

UDP (User Datagram Protocol)

  • بدون اتصال، سریع، اما بدون تضمین تحویل داده.
  • استفاده در استریم ویدیو، بازی‌های آنلاین، VoIP.

FTP (File Transfer Protocol)

  • پروتکلی برای انتقال فایل بین کلاینت و سرور.
  • دارای دو حالت:
    • Active Mode: ارتباط مستقیم کلاینت-سرور.
    • Passive Mode: مناسب برای عبور از فایروال‌ها.

7. وب‌سوکت (WebSocket)

WebSocket یک پروتکل ارتباطی دوطرفه و بلادرنگ است که به کلاینت و سرور اجازه می‌دهد بدون درخواست مکرر داده‌ها را ارسال کنند.

ویژگی‌های WebSocket

ویژگی توضیحات
ارتباط پایدار ارتباط دوطرفه و بلادرنگ بین کلاینت و سرور.
کاربردها چت آنلاین، بازی‌های تحت وب، به‌روزرسانی زنده محتوا.
